# تَلبِيَة — Talbiyah

## الوصف

في زحام الحج والعمرة، حيث كل خطوة تحمل معنى، يواجه ضيوف الرحمن تحديًا حقيقيًا: الوصول إلى المعلومات الصحيحة في الوقت المناسب وبالطريقة التي يفهمونها.

**تَلبِيَة** هو مساعد ذكي يعتمد على الذكاء الاصطناعي و البيانات اللحظية ، يهدف إلى إرشاد الحجاج والمعتمرين داخل الحرم بشكل فوري، دقيق، وإنساني.

يقوم النظام بـ:
- فهم **لهجة المستخدم العربية** (خليجي، مصري، شامي، تونسي , وغيرها)
- تحليل **حالة المستخدم** (هدوء، توتر، استعجال)
- تقديم **إرشادات واضحة ومطمئنة** حسب الوضع
- عرض **خرائط تفاعلية** مرتبطة بالموقف (بوابات، دورات مياه، مسارات خارج الزحام)

> الهدف: تحويل الرحلة من الحيرة إلى الطمأنينة.

---

## الفكرة الأساسية

بدلاً من تطبيق يعرض معلومات فقط،  
**تَلبِيَة يفهم المستخدم نفسه** — لغته، حالته، وسياقه — ثم يوجهه بثقة.

أمثلة:
- "متى أتحرك؟" → يحدد أفضل وقت بناءً على الزحام
- "أنا ضايع" → يهدئ المستخدم ويعطيه توجيه واضح

## طريقة التشغيل محليًا

### تشغيل الواجهة الخلفية

```bash
cd backend
pip install -r requirements.txt
python -m uvicorn app.main:app --reload
```
### تشغيل الواجهة الأمامية
```bash
cd frontend
npm install
npm run dev
```
### .env variables
- ELM_BASE_URL=
- ELM_API_KEY=
- ELM_MODEL=nuha-2.0
### الميزات الرئيسية
دعم اللهجات العربية المختلفة
استجابات ذكية تراعي حالة المستخدم
توجيه فوري داخل الحرم
خرائط تفاعلية مرتبطة بالسياق
تجربة بسيطة وسريعة مناسبة لبيئة الزحام

### حالة المشروع

⚠️ هذا المشروع هو نموذج أولي (Prototype)

تم تطويره خلال الهاكاثون لعرض الفكرة وإثبات إمكانية التنفيذ باستخدام الذكاء الاصطناعي.

لا يعتمد على بيانات حقيقية مباشرة من الحرم
الموقع الحالي يتم محاكاته لأغراض العرض
الهدف هو إظهار تجربة المستخدم وليس النظام النهائي

### الرؤية المستقبلية
ربط مباشر ببيانات الحشود الحقيقية
دعم الصوت (Speech-to-Text / Text-to-Speech)
تكامل مع الجهات الرسمية لإدارة الحشود
تحسين دقة تحديد الموقع داخل الحرم
دعم لغات إضافية للحجاج الدوليين
